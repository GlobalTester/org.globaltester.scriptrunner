Log Files

TestCampaigns and (if selected in the preferences) the single TestCases will have log files. For the steps of a TestCase, i.e. PreCondition, TestStep(s), and PostCondition, the line number of their logging beginning will be stored. If you open the log file for a step of a TestCase, it will highlight the line where its logging starts.

Due to the new GlobalTester APDU hover function, command APDUs can be easily read in the log files. They are syntax highlighted, and if you hover over it with your mouse, the APDU command is declared in its parts. You see the information

  • Which ISO Case command is used
  • CLA byte (indicates plain APDU or Secure Messaging)
  • INS (declares which command the APDU is)
  • P1 and P2 (possible protocols, etc.)
  • LC byte(s)
  • LE byte(s)

    • The response APDUs are also highlighted and thus can be easily spotted. Status words are extracted and interpreted.
