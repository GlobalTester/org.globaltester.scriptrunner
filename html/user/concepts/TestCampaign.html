TestCampaign

A TestCampaign is a collection of TestCases and their Executions. The collection of TestCases is static and is not to be changed after the TestCampaign is created. The Executions are the dynamic part of a TestCampaign, and you can start them as often as you like.

The TestCases that have been selected from their TestSpecification will be stored in the TestCampaign's subfolder "TestSpecification". There they can be accessed permanently, identically, and comparably to execute the TestCampaign.

When you execute a TestCampaign, you can choose the profiles its TestCases have assigned to. Only those TestCases that match the selected profiles will be executed.

Information of each execution is saved. It covers the start time of the execution, the duration, the result, a link to the log file and to the line the logging starts, and to the TestCase resource itself.
You can see and browse the Information, and open log files and TestCases in the TestCampaign editor.
You can also generate a report of your last execution.

The log files of the TestCampaign executions will be stored in the folder TestResults. If that option was set in the Preferences, the single log files for each TestCase will be stored there as well.
