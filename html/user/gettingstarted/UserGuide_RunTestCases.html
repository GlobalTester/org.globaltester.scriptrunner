User Guide: Run TestCases

In a few steps, you will be able to create a TestCampaign out of TestCases in your TestSpecification. In order to get a quick overview, you can also use the cheatSheet.

Create TestCampaign

In your TestSpecification, select one or more TestCases. Right-click your mouse. You can either choose "RunTest" or "Create TestCampaign". If you choose "RunTest", a TestCampaign will be created and run directly.

Run TestCampaign

You can (re-) run your TestCampaign by clicking the "Execute" button.

Generate Report

You can generate a report of your run by clicking the "Generate Report" button. A folder browser will open where you can choose the folder you want the report to be saved in.

View TestCases and TestSteps

In the TestCampaignEditor in the column "TestCase/TestStep" the whole TestCampaign structure will be shown as a tree. You can double-click with your mouse on the single TestCases or TestSteps, and the TestCase will open in an editor.

View LogFiles

In the TestCampaignEditor in the column "TestCase/TestStep" the whole TestCampaign structure will be shown as a tree. You can open a LogFile to an item if you right-click on it with your mouse, then choose "Show LogFile". If you select the TestCampaign, the log file for it including all testcases and teststeps will open in an editor. If you have selected "Add separate logfile for each testcase" in the GlobalTester Preferences and select a TestCase, the log file for this TestCase will open. If you have selected "Add separate logfile for each testcase" in the GlobalTester Preferences and select a TestStep, the log file for this TestCase will open with the line number of the beginning of the TestStep highlighted.

Browse TestCampaign Executions

Status of TestCase Execution in TestCampaign Execution

In the TestCampaignEditor in the column "Status" the status of the TestCase execution will be shown. It is either "FAILURE" (red), "WARNING" (yellow), or just green.
