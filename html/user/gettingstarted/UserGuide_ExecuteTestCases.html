Execute TestCases

In a few steps, you will be able to create a TestCampaign out of TestCases in your TestSpecification and evaluate them. In order to get a quick overview, you can also use the Cheat Sheet.

Setup Test Environment

Make sure your card reader is connected to you computer.

If you have to connect it first and GlobalTester 3 is running, GlobalTester 3 will not be able to recognize and embed it into its functionality, so please start GlobalTester 3 again.

Place your card on the reader.

Import your TestSpecification

If not already done, import one or more TestSpecification(s).

Create and execute TestCampaign

There are two possibilities to create a TestCampaign:

  1. Create a TestCampaign and execute it, or
  2. Select TestCases to execute (this will create a TestCampaign as well).

Create a TestCampaign and execute it

In the TestSpecification(s) you have chosen, select the TestCases you want to execute. Right-click your mouse, and choose "Create TestCampaign". A TestCampaign will be created and opened in the TestCampaign Editor. It consists of your selected TestCases, the execution states, the log files documenting the test results, and the subset of the TestSpecification(s) you have chosen (i.e. the TestCases you have chosen).

You can (re-) execute your TestCampaign by clicking the "Execute" button.

Select TestCases to execute

In the TestSpecification(s) you have chosen, select the TestCases you want to execute. Right-click your mouse, and this time choose "RunTest".

A TestCampaign will be created and executed directly. Of course you can execute it again by clicking the "Execute" button.

Evaluate Log File(s)

Each execution of a TestCampaign will have an own log file. There, the logging of all executed TestCases will be reported. You can see the communication between the card and the reader.

If you have selected "Add separate logfile for each testcase" in the Logging-Preferences, each executed TestCase will have its own log file.

You can easily access the log files:

  • In the TestCampaign editor, you can see a tree view displaying the last execution of the TestCampaign structured in its TestCase and Test Steps. Right-click with your mouse on the root of the tree view which corresponds to the last execution of the TestCampaign, and choose "Show log file".

    The log file of the last execution of the TestCampaign will open.
  • If you have selected the option for single log files, you can open it by right-clicking on a TestCase and choosing "Show log file".
  • If you right-click on an ActionStep (i.e. PreCondition, TestStep, or PostCondition), the log file (for the last execution of the TestCampaign or of the single TestCase if selected) will open with the line highlighted in which the log of the ActionStep begins.

If you have a "Failure" reported in the TestCampaign editor, you can find the error in the TestCase execution by looking for "@FailureID1", "@FailureID2", etc. The lines in the log file where they appear will have a red marker. When you click on it, the line where the failure is reported will be highlighted in the opening log file.

Usually "@FailureID1" is the essential source of error. For example, if you have no card placed on you reader, it will be the missing "card" reference.

The APDU commands are syntax highlighted, and if you hover over it with your mouse, the APDU command is declared in its parts. You see the information

  • Which ISO Case command is used
  • CLA byte (indicates plain APDU or Secure Messaging)
  • INS (declares which command the APDU is)
  • P1 and P2 (possible protocols, etc.)
  • LC byte(s)
  • LE byte(s)

The response APDUs are also highlighted and thus can be easily spotted. Status words are extracted and interpreted.

They can be checked, and be another source for a "Failure" report.

Generate Report

You can generate a report of your execution by clicking the "Generate Report" button. A folder browser will open where you can choose the folder you want the report to be saved in.

The test report will comprise

  • Test summary: specifies the tilte and the version of the TestSpecification.
  • Test setup
  • Test statistics: lists the date of the execution, execution time, and the numbers of passed and failed tests.
  • Overview of executed test cases: lists all TestCase IDs, their Description, the time of the execution, and the result (reported as "Passed", "Failure", or "Unknown")
  • Test case results
